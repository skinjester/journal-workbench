## [2013-02-05 09:21] skinjester.com work

testing with high latency/low bandwidth network reveals an edge case in which typekit font will always fail to load and will fire the inactive event on each page load. Because the entire page is hidden and transitions in upon typekit firing an active or inactive event, this creates a delay in which nothing is visible on the page until the event is heard. I think that if typekit fails to load the font on the first attempt, it shouldnt bother making the attempt for subsequent page loads. How would it be possible to know this though? I previously had a block of code that set and listened for a cookie. perhaps that could be used to set a flag (success/fail) If the cookie isnt set, then attempt to load typekit. set the cookie to reflect the state of the font load. On subsequent page loads check the cookie... except this creates further complication. what if I return to the page at a later time. the cookie is still set. or perhaps cookies have been disabled... thats not a good solution. Instead, I think typekit should attemptto load fonts asynchronously. How would that work though? Right now, everything is dependent upon the font being ready or not ready...
