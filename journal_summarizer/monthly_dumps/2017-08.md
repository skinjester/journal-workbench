after discussion with the team here are a couple of things I can consider to be anchors:
1. Search Query can be presented as a modal dialog
2. Search results can be surfaced within the Query Modal (but is a UX requirement to think that though
3. Search Results pages aren't the same as Archive results pages
	- broader immediate requirements for search, namely each results page needs to be addressable as a destination through a URL
	- as a Tier-3 feature, it is worth re-examining how Archive Results and Search Results can be generalize to be simply "Results"
4. Do not include boilerplate queries or links in the Query phase
	- was asked to do so, but universal pushback against doing so

The design document I am standing up deliberately seperates the query phase from the results phase. If its felt that adding results to the query phase is the preferred choice, I'm not against doing so either. The larger issue is presenting those outcomes more smoothly. The current implementation gests us there functionally, but is an awkward presentation.
