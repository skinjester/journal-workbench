## [2018-01-03 14:39] eso

going back over the beginner tutorial

each SSP template is made of several template files.
Every template must have at minimum, a region file (usually named site.region) and a template config file, named template.conf

Templates fall into 3 categories
1. site-wide files
2. collection-specific files
3. block-specific files

site.region
	- defines layout of template pages
	- typically used as the global site wrapper
		+ containing site header
		+ site footer
		+ sidebar
	- typically used as the default layout
	- composed of HTML and JSON-T

template.conf
	- uses JSON to config elements within the template
	- name your template
	- specify layouts
	- add navigation sections
	- specify stylesheets to be rolled up into site.css

/assets
	- static design assets
		+ images
		+ fonts
		+ icons

/blocks
	- block files
	- JSON-T "partials"
		- reusable code that can be included elsewhere in your JSON-T
		- like a macro, I suppose

/collections
	- collection files
		- (collection).list
			+ list view of the collection is default view
			+ sets up structure of how a list will look
			+ HTML/JSON-T
		- (collection).item
			+ sets up striucture of how single post will look
			+ HTML/JSON-T
		- (collection).conf
			+ tells squarespace platform how each collection is configured
			+ JSON

/pages
	- static page files
		- (static).page
		- (static).page.conf

/scripts
	- javascript files
		- site.js

/styles
	- stylesheets
		- styles.css
		- styles.less

(root)
	- sitewide files
		- site.region
		- template.conf

		+ Incorporate tropes and patterns from text-driven RPG's
