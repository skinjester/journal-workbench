MESA MENU DESIGN SPECIFICATION

The MESA Menu directs customers to unique MESA URL's throughout the campaigns lifecycle. These URL's are assumed to be dedicated Promotional Pages for DLC and Chapter updates

The MESA Menu is a component that is intended for use on the following page types:
	- Chapter Page
	- DLC Page
	- Custom Page built from Chapter Page or DLC Page Template

This component may also have value if used as a sidebar widget on the following page types:
	- Home Page
	- News Page
	- Article Page

This component includes clickables that link out to dedicated MESA content throughout the campaign

The arrangement of the clickables at any viewport width implies progression (linear/radial)

The sizing/theming of the clickables signifies differences between Chapter, DLC or Event Promotion

The clickables may be shown in an unlocked or locked state
	+ The unlocked state may refer to any available URL, including the current page
	+ The locked state isn't navigable

This component incorporates a background asset which may be specified or overriden when placed on a page

taking a closer look at optical flow implementation, especially with respect to the framebuffering features I've added, although not yet fully exploited. I've shown in previous experiments though, how to maintain an image queue - just a collection of images captured asynchrously and playing back in realtime.

The idea is to compute optical flow from these images and use that information to cause the images to blend smoothly into one another. The theory is that this effect should look similar to some of the after effects work I've previously done on sequences where the framerate is altered and the "pixel blending" algorithm is used, which is also based on optical flow.

The reason its important to process this way:
1. visual effect is pretty great
2. earlier experiments w asynchronous buffering and sequential playback of deepdream content wasn't really tghought through fully - but still - it was very noisey - very jumpy. Not uninteresting, but not the smoothness I've been working to achieve.
